How to read this set

  1. Technical v2 for the stack-level decision.
  2. Competitive v2 to pressure-test category and substitution risk.
  3. GTM v2 for who buys, from what budget, and what proof is still missing.
  4. Skeptical memo + risk register + pilot checklist to turn the analysis into actual decision gates.

Note: this is a static GitHub Pages site with a client-side password gate. That is fine for light private sharing, but not real high-security protection.
